# Public data to train chatbots

## Datasets

-   **AmFam Chatbot Intent Dataset (ACID)**: Contains 174 unique intents related to topics driven by customers contacting an insurance company. Each intent represents a particular course of action for the chatbot. The intent prediction dataset was collected from past interaction of customers with service representatives at American Family Insurance. You can download the dataset from  [here](https://github.com/AmFamMLTeam/ACID).
-   **InsuranceQA Corpus**: This dataset contains 14,022 question-answer pairs collected from insurance-related forums.
-   **Insurance Chatbot Dataset**: This dataset contains 10,000 customer queries and their corresponding responses from an insurance chatbot.
-   **NOAA Datasets**: Google Cloud Public Datasets offers more than 100 high-demand public datasets through BigQuery that helps insurers in these sorts of data “mashups.” You can find more information about the datasets  [here](https://cloud.google.com/blog/topics/financial-services/insurers-use-noaa-datasets-for-predictive-analytics).
-   **Ubuntu Dialogue Corpus**: Consists of almost one million two-person conversations extracted from the Ubuntu chat logs, used to receive technical support for various Ubuntu-related problems. You can download the dataset from  [here](https://www.kaggle.com/rtatman/ubuntu-dialogue-corpus).
-   **AppleSupport, AmazonHelp, Uber_Support, Delta and SpotifyCares**: These are customer support chatbots that were implemented and trained on publicly available data. You can download the dataset from  [here](https://github.com/ldulcic/customer-support-chatbot).
-   **Customer Support Datasets for Chatbot Training**: This dataset features human-computer data from three live customer services representatives who were working in the domain of travel and telecommunications. It also contains information from airline forums that were featured on TripAdvisor.com. You can download the dataset from  [here](https://imerit.net/blog/14-best-chatbot-datasets-for-machine-learning-all-pbm/).
- [Cornell Movie Dialogs Corpus](https://www.cs.cornell.edu/~cristian/Cornell_Movie-Dialogs_Corpus.html): A large metadata-rich collection of fictional conversations extracted from raw movie scripts.
- [Stanford Question Answering Dataset (SQuAD)](https://rajpurkar.github.io/SQuAD-explorer/): Contains 100,000+ questions posed by crowdworkers on a set of Wikipedia articles, where the answer to every question is a segment of text, or span, from the corresponding reading passage.
- [MultiWOZ (Multi-Domain Wizard-of-Oz)](https://multiwoz.org/): A dataset for developing models for multi-domain, task-oriented dialogue systems.
- [CoQA (Conversational Question Answering)](https://stanfordnlp.github.io/coqa/): Contains 127,000+ questions with answers, obtained from 8,000+ conversations.
- [RASA](https://rasa.com/): An open-source platform for building chatbots and virtual assistants. It includes some sample conversations that can be used for training.
